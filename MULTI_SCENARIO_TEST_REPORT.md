## 🎯 场景适配策略

### 1. 会议场景（多人）
- **自动检测说话人数** - pyannote 自动识别
- **记录发言占比** - 识别主导者和沉默者
- **话题转换检测** - 基于关键词频率变化

### 2. 朗读/话术（单人）
- **语速稳定性** - 朗读通常语速均匀
- **停顿模式** - 预准备内容有规律停顿
- **填充词少** - 话术考核应少"啊""嗯"

### 3. 电话对话（双人）
- **轮流模式** - 典型的一问一答
- **响应延迟** - 测量对话流畅度
- **打断检测** - 记录抢话行为

### 4. 背景噪音处理
- **当前限制**: 无法区分"目标说话人"和"背景人声"
- **改进方向**: 
  - 声纹识别（锁定目标说话人）
  - 信噪比分析（过滤低能量背景声）
  - 语义连贯性（背景人声通常与主题无关）

## 📝 设计验证结论

### ✅ 已实现

1. **客观指标完整** - VAD、韵律、填充词、说话人分离
2. **原始内容保留** - 不优化、不加工、不修正
3. **多场景支持** - 自动检测说话人数，无需用户指定
4. **时间戳分段** - 每句话精确到秒
5. **关键词提取** - 自动识别话题

### ⚠️ 待改进

1. **背景噪音区分** - 无法识别"无关背景人声"
2. **朗读 vs 对话分类** - 需要额外分类器
3. **摘要质量** - 启发式方法质量一般，需 LLM
4. **长音频处理** - 3 小时文件需分段处理
